# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in this project, please report it confidentially via email to: [sorrentinogiuse@icloud.com](sorrentinogiuse@icloud.com). To ensure your message is not marked as spam, use the exact subject line: **[Security Report] - Library Name**. Please avoid public disclosures such as GitHub issues or discussions.

## Handling Process

- We acknowledge receipt of your report within 48 hours.
- We will investigate and work on a fix as soon as possible.
- Security patches will be released promptly, with updates communicated.

## Supported Versions

Security fixes are provided only for the release 3.x.x and 2.x.x of the library.

## What We Consider Security Issues

- Remote code execution
- Arbitrary code execution via malicious input
- Leakage or exposure of sensitive data
- Denial of Service caused by resource abuse or overload

Bugs, crashes due to malformed input during development, or improper use of the library are not considered security vulnerabilities.

## Risk Considerations

This library does not handle sensitive data nor exposes significant attack surfaces; therefore, we do not anticipate high risks. However, we take all reports seriously and will carefully evaluate any potential issues.

## Disclaimer

The project is provided "as-is" without any warranties. Users should carefully assess their usage in their specific context.
